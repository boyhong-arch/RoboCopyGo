Robocopy简介

Robocopy（Robust File Copy）是微软在 Windows Server 2003 Resource Kit Tools 中提供的命令行程序，后来 Vista、Win7 等后续系统已经自带。与传统复制功能相比，Robocopy 提供了更强大的文件管理能力。

传统复制局限性

• 要么全有要么全无的复制方式

• 大目录拷贝用时很长且不能中断

• 不支持权限属性复制

• 不支持增量备份

• 无法过滤特定类型文件

Robocopy 优势

• 支持断点续传，复制可中断后继续

• 保持文件权限、属性和时间戳

• 支持镜像备份和增量备份

• 可过滤文件类型和设置文件大小限制

• 支持多线程复制，提高效率

注意：Robocopy 是系统自带工具，无需额外购买商业软件许可，是替代同类商业软件的优秀选择。

基本语法

ROBOCOPY source destination [file [file]...] [options]
源目录 :: 源目录(驱动器:\路径或\\服务器\共享\路径)
目标目录 :: 目标目录(驱动器:\路径或\\服务器\共享\路径)
文件 :: 要复制的文件(名称/通配符: 默认为 "*.*")
提示：Robocopy 支持网络路径，可以直接复制到文件服务器共享文件夹。

参数详解

复制选项

/S - 复制子目录，但不复制空的子目录

/E - 复制子目录，包括空的子目录

/LEV:n - 仅复制源目录树的前n层

/Z - 在可重新启动模式下复制文件

/B - 在备份模式下复制文件

/MT[:n] - 使用n个线程进行多线程复制

文件选择

/A - 仅复制具有存档属性集的文件

/M - 复制具有存档属性的文件并重置属性

/XF - 排除指定文件

/XD - 排除指定目录

/MAXAGE - 设置最大文件存在时间

/MINAGE - 设置最小文件存在时间

高级功能

/MIR - 镜像目录树(等同于 /E 加 /PURGE)

/MOV - 移动文件(复制后从源中删除)

/PURGE - 删除源中不存在的目标文件

/COPYALL - 复制所有文件信息

/MON - 监视源目录变化

/MOT - 按时间监视源目录

常用参数组合

参数组合 功能描述 适用场景
/E /COPYALL 完全复制所有子目录和文件信息 完整备份，需要保留所有属性
/MIR 创建完全相同的镜像副本 同步备份，保持源和目标一致
/S /Z 复制子目录(非空)并支持断点续传 网络传输，可能中断的大文件复制
/E /MT:16 多线程复制所有子目录 提高大数量文件复制速度

实用示例

实例一：选择性复制特定文件类型

robocopy d:\work e:\back *.txt *.doc *.bmp *.tif /s

实现效果：将d:\work文件夹下的TXT、DOC、BMP、TIF文件复制到e:\back文件夹，包括非空子目录。

参数说明：/s表示复制子目录，但不复制空的子目录。

举一反三：如果要将所有文件包括空文件夹全部复制，使用命令：robocopy d:\work e:\back /e

实例二：排除垃圾文件

robocopy d:\work e:\back /e /xf *.tmp *.bak

实现效果：复制所有文件但排除TMP和BAK垃圾文件。

参数说明：/xf为指定不执行复制操作的文件类型，后面的文件名支持通配符。

实例三：完全镜像备份

robocopy d:\work e:\back /mir

实现效果：创建完全相同的镜像备份，删除目标文件夹中存在但源文件夹中没有的文件。

参数说明：/mir等同于/e和/purge，表示镜像目录树。

实例四：保留NTFS权限

robocopy d:\work e:\back /e /copyall

实现效果：复制所有文件及NTFS权限、所有者等安全信息。

参数说明：/copyall表示复制所有文件信息(等同于/COPY:DATSOU)。

实例五：自动监控备份

robocopy d:\work e:\back /e /copyall /mot:1 /mon:2

实现效果：监控源文件夹，当1分钟内发生2处以上修改时自动执行备份。

参数说明：/mot指定检查时间(分钟)，/mon指定最少修改文件数。

高级技巧：可以将Robocopy命令保存为批处理文件(.bat)，然后通过Windows计划任务实现定时自动备份。

注意事项

1. 使用 /PURGE 或 /MIR 时，Robocopy 将跳过"系统卷信息"目录

2. 低可用空间模式与 /MT、/EFSRAW、/B 和/ZB 不兼容

3. 多线程复制(/MT)与 /IPG 和 /EFSRAW 选项不兼容

4. 使用 /LOG 选项重定向输出以便获得最佳性能

5. 首次使用建议先使用 /L 参数试运行，只列出要执行的操作而不实际复制
